### 需求

1、手写Markdown发布文字博客，有最基本的目录、标签、文章列表功能，排版简单，暂时不需要冗余功能，图片存储的需求极小
## 选型与原理

### 选型

| 模块            | 技术                                                                          | 说明             |
| ------------- | --------------------------------------------------------------------------- | -------------- |
| **静态生成器**     | **Hugo**（主题：[not-much](https://themes.gohugo.io/themes/not-much/), 极简/易自定义） | 性能相较于Hexo较高    |
| **托管平台**      | **Vercel**                                                                  | 免费、全球 CDN、自动部署 |
| **DNS + CDN** | **Cloudflare**                                                              | 国内回源加速，防 DDoS  |
| **版本控制**      | **Git + GitHub**                                                            | 私有仓库           |

总成本：域名费用 + 0元 = 域名费用 
### 原理

1. 静态网站生成器
	作用：将你的md文档直接按照模板生成html
2. 托管平台
	作用：帮你托管已经生成好的html
3. DNS+CDN
	作用：由于某些限制，我们访问不了我们的托管平台，所以你需要找一个可访问的中间平台作为传话人
4. 版本控制
	作用：存放代码的仓库

## 本地环境搭建
### 安装git（略）

### 安装hugo
1. 地址：
	https://github.com/gohugoio/hugo/releases
2. 添加环境变量：
	将“hugo.exe”所在的路径添加到 系统环境变量的Path中
3. 阅读hugo的quickstart： https://gohugo.io/getting-started/quick-start/


### 创建hugo项目骨架
```
hugo new site <博客名>
```

### 选择Hugo主题

极简风格主题：
https://themes.gohugo.io/themes/not-much/
### 本地运行Hugo

步骤：
1. 参考主题的官方文档先跑起来
2. 了解主题架构，文件大致作用（可选）
3. 了解主题提供的功能、参数（可选）
4. 了解hugo server启动命令后到浏览器正常浏览博客的全过程（可选）
5. 主题代码自定义修改（可选）
### 本地测试

发布一篇文章测试整个本地的流程

## 项目的部署

**前提条件：需要科学↑网环境**

### 部署到vercel

1. 注册vercel
2. 从github导入项目
3. 添加环境变量：HUGO_VERSION = 0.152.2  
	0.152.2作为参考，具体与你本地hugo版本保持一致
4. 一键部署

## 网络加速与代理

**前提条件：需要购买一个域名**

### Cloudflare加速

目的：利用cloudflare免费提供的方案，实现DNS解析和CDN分发功能，实现国内能流畅访问

步骤：
1. 将你的域名控制台中的DNS服务器地址，修改为cloudflare提供的DNS域名地址
2. 为vercel提供的域名配置代理
	DNS配置 CNAME, 指向vercel提供的域名
3. 去到velcel项目管理页面，将自己的域名添加为白名单

**此时，即可访问自己的域名去到自己的博客主页了。**

## 发布文章的流程
1. 创建文章：
	hugo new posts/name.md
2. 编写md
3. 推送远程
```md
	git pull origin main
	git add .
	git status
	git commit -m "new post"
	git push origin main
```
